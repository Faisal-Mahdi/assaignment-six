Discuss the scope of var, let, and const

var declarations are globally scoped or function scoped while let and const are block scoped. var variables can be updated and re-declared within its scope; let variables can be updated but not re-declared; const variables can neither be updated nor re-declared. They are all hoisted to the top of their scope. But while var variables are initialized with undefined, let and const variables are not initialized. While var and let can be declared without being initialized, const must be initialized during declaration.

What do you mean by REST API?

REST API, which stands for Representational State Transfer Application Programming Interface, is a set of architectural principles and constraints for designing and interacting with web services. It is an approach to building web services that are scalable, stateless, and designed to work over HTTP (Hypertext Transfer Protocol).
Key characteristics and concepts of REST API include:
1. Resources: In a RESTful API, everything is treated as a resource, which can be a data object, a service, or even a concept. Resources are identified by unique URIs (Uniform Resource Identifiers).
2. HTTP Methods: RESTful APIs use standard HTTP methods for performing operations on resources: - **GET**: Retrieve data from the server. - **POST**: Create a new resource on the server. - **PUT**: Update an existing resource on the server or create it if it doesn't exist. - **DELETE**: Remove a resource from the server. - And others like PATCH, HEAD, OPTIONS, etc.
3. Statelessness: REST APIs are stateless, meaning that each request from a client to a server must contain all the information needed to understand and process the request. There should be no reliance on the server's previous state.
4. Uniform Interface: REST APIs have a uniform and consistent interface, which simplifies the client's interaction with different resources. This includes the use of standard HTTP status codes (e.g., 200 OK, 404 Not Found) and standard conventions for resource naming.
5. Representation: Resources can have multiple representations (e.g., JSON, XML, HTML) to convey information to clients. Clients can specify their preferred representation in the request headers.
6. Stateless Communication: Each client request to the server is independent, and the server does not retain any information about the client's state between requests. Any necessary session state should be managed on the client side.
7. Layered System: REST allows for the use of intermediary components (e.g., load balancers, caches, gateways) to improve system scalability and performance. Clients are unaware of the existence of these intermediaries.
8. Client-Server Architecture: REST separates the concerns of the client and server, allowing them to evolve independently. This separation enhances scalability and flexibility.
9. Idempotence: Some HTTP methods, like GET and PUT, are idempotent, which means that making the same request multiple times will have the same effect as making it once. This property is important for ensuring predictability and safety of operations.
REST APIs are widely used for building web services and are the foundation of many modern web applications and mobile apps. They provide a standardized and flexible way for different systems to communicate and exchange data over the internet.
